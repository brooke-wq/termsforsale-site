Five GHL workflows · Three n8n workflows

Each is single-purpose and triggered by a specific event. You can trace any buyer's path through the system using their contact history.

WF01

Buyer Intake & Scoring

Trigger: buyer:new tag OR form submit

Welcome SMS + email, qualifier nudge, 14-day re-engage, then A/B/C tier scoring after intake call.

WF02

Deal Match & Send

Trigger: deal:new-inventory tag

Tier A → now. Tier B → +1h. Tier C → +4h. SMS + email. 24h later: if buyer engaged → task to Eddie.

WF03

Close & Recycle

Trigger: stage = Closed

Upgrades buyer to VIP + Tier A. Increments 12-month deal count. Creates assignment-fee confirmation task.

WF04a

90-Day Dormant Check

Trigger: daily at 8am AZ

Sweeps buyer:active contacts with Last Touch > 90d. Moves them to dormant.

WF04b

Quarterly POF Re-Verify

Trigger: daily at 9am AZ

Pings buyers whose Re-verify Due Date hit. 5-day reply window. No reply → dormant.

n8n

Match Engine + Notion Bridge + Helper

Triggers: webhook + 10-min cron

Notion bridge polls every 10 min. Match engine fires on each deal. Helper does +1 math that GHL can't.

End-to-end: one deal's journey

Inbound

  • Minute 0 — Brooke marks deal Actively Marketing in Notion
  • ~10 min — Notion bridge picks it up, POSTs to match engine
  • <1s — Match engine fetches active buyers, filters on asset class ∩ market ∩ price
  • <1s — Writes "latest deal" custom fields onto each matched buyer, applies deal:new-inventory tag

Outbound

  • <1 min — Tier A buyer gets VIP-labeled SMS + email
  • +1h — Tier B buyer gets the same
  • +4h — Tier C buyer gets the same
  • +24h — If any buyer replied → Eddie gets task "call within 2 hours"
  • Close — WF03 fires; buyer auto-promoted to VIP, deal counter bumped
No operator touch in the middle. From "Actively Marketing" to buyer SMS is fully automated.

When it breaks

How to route anything that isn't firing — and how the system recovers.

Historical note. A "WF02 SMS gap" used to affect API-applied tags intermittently. It was resolved 2026-04-20 by migrating WF02's trigger from "Contact Tag Added" to an inbound webhook that n8n POSTs to directly. If you hear the old gap referenced in a Slack thread or SOP, it's historical.
